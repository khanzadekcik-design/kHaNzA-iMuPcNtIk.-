Topologi & Arsitektur

Dasar & contoh
Topologi = bagaimana perangkat saling terhubung. Arsitektur = model layanan & kontrol (Client-Server, Peer-to-Peer, dsb.). Pilih arsitektur sesuai skala, keamanan, dan manajemen.

Jenis Topologi

  • Bus — kabel tunggal, sederhana tapi cepat penuh untuk banyak node.
  • Star (Bintang) — semua perangkat ke hub/switch; mudah kelola dan skalabel.
  • Ring (Cincin) — node tersusun melingkar; cocok untuk token passing.
  • Mesh (Jala) — redundansi tinggi; setiap node ke beberapa node lain.
  • Tree (Pohon) — hirarkis, gabungan star & bus.
  • Hybrid — kombinasi topologi di atas.

Arsitektur Jaringan

  • Client-Server — server pusat sediakan layanan. Baik untuk organisasi besar.
  • Peer-to-Peer (P2P) — semua node setara; cocok untuk jaringan kecil atau berbagi sederhana.

Tips memilih

  1. Skala jaringan: kecil & sederhana → P2P; banyak pengguna & layanan → Client-Server.
  2. Keamanan & manajemen: Client-Server lebih mudah dikontrol.
  3. Redundansi: gunakan mesh/dual-homed link untuk layanan kritis.

Subnetting, CIDR & VLSM

Konsep singkat + alat

Subnetting

Membagi jaringan besar menjadi beberapa jaringan kecil (subnet) agar ip lebih efisien dan broadcast terkontrol.

CIDR

Classless Inter-Domain Routing — notasi /prefix (contoh: 192.168.1.0/24) menunjukkan jumlah bit jaringan. Lebih fleksibel dibanding kelas A/B/C.

VLSM (Variable Length Subnet Mask)

Memungkinkan subnet dengan ukuran berbeda sesuai kebutuhan host setiap subnet. Efisien untuk alokasi alamat.

Gunakan panel sebelah kanan untuk mencoba kalkulator CIDR dan perencana VLSM.
